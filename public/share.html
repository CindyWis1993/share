CSS、SVG-轨迹动画

（一）为什么选择分享轨迹动画？

首先，前端开发中经常需要实现各种动画效果，特别是轨迹动画；比如世界杯射门小游戏（球类的弧线轨迹运动），捕鱼小游戏（鱼类元素的游动轨迹）、各类活动的飘落或震动轨迹等（活动红包等），因此这次就选择最常见的轨迹动画跟大家分享。

（二）为什么选择用CSS、SVG实现？

其实JS结合Canvas或者Flash可以实现更加复杂多变的轨迹动画效果，但是结合我们日常开发需求，css更为常用；而且我们官网H5已经开始使用SVG图片，因此结合实际我选择总结了CSS、SVG实现轨迹动画（其实还有很多方式，比如GIF&帧动画；GIF现在已经很少用于动画展示了，图像通常很大且清晰度较差；帧动画这里也不做分享，通常复杂的轨迹动画若用帧动画实现同样需要非常多的图片，且不易修改）。

常用的轨迹动画主要分为三种：直线轨迹动画、曲线轨迹动画、不规则/复杂轨迹动画，下面就一 一来看下实现方式：

1、直线轨迹动画

直线轨迹动画是轨迹动画中最为简单常用的一种，通常可以直接两点定线来实现；如果需要做速度的调整，则直接改变运动的时间即可，因此首选css方式。通常，css直线轨迹动画我们会选择用top/left或者translate的变化实现，哪种更优？
top/left花了大量的时间去绘制每一帧。所有的css都是在CPU上计算的。
translate完全的使用GPU，效率更高，浏览器可以优化大量绘制和组合的时间。 

                
    .straightLine {
        animation: straightLine 2s alternate infinite;
    }
    @keyframes straightLine {
        0% { transform: translate(0, 0); }
        30% { transform: translate(100px, 0); }
        60% { transform: translate(200px, 100px); }
        100% { transform: translate(300px, 100px); }
    }
(0, 0) (100, 0) (200, 100) (300, 100)

rotate实现轨迹动画，可以利用容器嵌套，外层容器旋转实现，但只适用于轨迹为圆形或圆形的一部分的情况：

实际开发中，基本不可能只需要实现简单的直线或者圆弧运动，经常会出现不规则的弧线运动，比如椭圆弧形运动，这时候就需要两个方向运动结合。
其原理是：对一个物体来说，运动永远是受X轴以及Y轴的力控制，因此单独对一个物体无法进行改变，只能达到直线轨迹的效果，要想实现双向运动实现曲线轨迹，只能再增加一个元素，使其互相影响，一方负责X轴运动，另一方负责Y轴运动实现可以实现椭圆等特殊的轨迹效果（JS可以实现单一物体两方向不同速率实现弧形运动），但是弧度的计算比较复杂，较难调整：


具体如何实现呢，首先为了创建一条不成直线的路径，我们希望X轴和Y轴的运动速度不同步，因此需要对X轴跟Y轴设置两种计时函数， 所以，分别对X轴跟Y轴使用ease-in、ease-out，以下是对应的描述：

                
    .moveLine-1 {
        animation: moveLineX 2.5s infinite ease-in;
    }
    .moveLine-2 {
        animation: moveLineY 2.5s infinite ease-out;
    }
    .moveLine-3 {
        animation: moveLineX 2.5s infinite ease-in;
    }
    .moveLine-3::after {
        animation: moveLineY 2.5s infinite ease-out;
    }
    @keyframes moveLineX {
        50% {
        animation-timing-function: ease-in;
        transform: translateX(100px);
        }
    }
    @keyframes moveLineY {
        50% {
        animation-timing-function: ease-out;
        transform: translateY(100px);
        }
    }

从上面的描述中我们注意到ease-in、ease-out等效果的实现，实际上是用了贝塞尔曲线（cubic-bezier），因此我们需要先了解贝塞尔曲线是什么： 贝塞尔百度百科

一阶贝塞尔曲线只是一条两点之间的直线： 二阶贝塞尔：
三阶贝塞尔（css曲线效果）：

在p0p1、p1p2、p2p3等等的起点到控制点再到终点的连线中，每段连线都被分割成了两部分（仔细看动图中的黑色、绿色、蓝色圆点），各段连线中两部分的比值都是相同的，比值范围是0到1，而这个比值就是t。


CSS3动画速度的控制通过三次贝塞尔曲线函数实现，定义规则为:
cubic-bezier (x1, y1, x2, y2)

P0：默认值 (0, 0)
P1：动态取值 (x1, y1)
P2：动态取值 (x2, y2)
P3：默认值 (1, 1)

关于三次贝塞尔曲线最直接的理解是，将以一条直线放在范围只有 1 的坐标轴中，并从中间拿出两个点来拉扯（X 轴的取值区间是 [0, 1]，Y 轴任意），最后形成的曲线就是动画的速度曲线。

但是贝塞尔曲线实际应用于动画开发中时，由于计算比较复杂（大学高数的贝塞尔曲线不要忘光了，了解更多->图形算法：贝塞尔曲线），一般很难完全还原本身设计的动画轨迹，因此可以借助一些辅助工具实现： 点击就送可视化贝塞尔曲线
